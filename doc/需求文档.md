# AgentHub - 多Agent协作平台 需求文档

## 项目概述

AgentHub 是一个多Agent协作平台，采用IM聊天作为核心交互范式。用户像使用飞书/微信一样，通过新建对话、发送消息的方式与不同AI Agent进行交互。每个Agent就是一个"聊天对象"，平台同时接入主流Agent平台，通过统一适配器层屏蔽API差异，并支持用户自建Agent。

### 技术架构概览

```
浏览器(React) ←→ (WebSocket + REST) ←→ Go后端 ←→ (WebSocket) ←→ 本地守护进程 ←→ (stdin/stdout) ←→ CLI Agents
```

- **前端**：React + TypeScript，浏览器访问
- **后端**：Go，提供REST API + WebSocket服务
- **数据库**：PostgreSQL（用户、对话、消息、Agent配置）
- **通信协议**：WebSocket（实时消息/流式输出）+ REST（CRUD操作）
- **本地守护进程**：运行在用户电脑上，主动WebSocket连接后端，扫描并管理本地CLI Agent进程
- **鉴权**：简单用户名密码，多用户支持

## 核心功能

### 1. IM聊天式交互（核心体验 — P0）

#### 1.1 对话管理

| 功能 | 说明 |
|------|------|
| 对话列表 | 左侧会话列表，支持新建/置顶/归档/搜索，按最近活跃排序 |
| 多会话并行 | 同时开启多个对话窗口，分别与不同Agent交流不同任务 |
| 用户鉴权 | 简单用户名密码登录，每个用户有独立的对话和Agent配置 |

#### 1.2 聊天模式

| 模式 | 说明 |
|------|------|
| 单聊模式 | 1v1与单个Agent对话，适合明确任务（如"用Claude Code写一个React组件"） |
| 群聊模式 | 一个对话中包含多个Agent，通过@指定，由Orchestrator自动协调分工 |

#### 1.3 消息系统

| 功能 | 说明 |
|------|------|
| 消息类型 | 文本、代码块（语法高亮）、图片、文件附件、网页预览卡片、Diff视图卡片 |
| 消息操作 | 回复、引用、重新生成、复制代码、展开预览 |
| 流式输出 | Agent回复逐字流式展示，通过WebSocket实时推送 |
| 上下文管理 | 聊天历史自动作为上下文传递给Agent；支持Pin消息作为长期上下文注入 |

#### 1.4 Pin上下文机制

- 用户可右键或点击消息选择"Pin"
- 被Pin的消息内容自动注入到后续所有发给Agent的请求上下文中
- 用于保存关键需求、技术约束、重要决策等长期有效信息
- 支持取消Pin

---

### 2. Orchestrator主Agent协调器（P0）

Orchestrator是群聊模式下的主Agent，采用**主从关系**协调其他子Agent。

#### 2.1 核心能力

- **LLM驱动**：复用本地已安装的Claude Code CLI作为LLM引擎，通过特殊的Orchestration System Prompt驱动
- **智能意图理解**：分析用户消息，理解任务目标和约束
- **任务拆解**：将复杂任务分解为可独立执行的子任务
- **并行调度**：将子任务同时分派给多个合适的子Agent执行
- **结果聚合**：收集所有子Agent的产出，合并为连贯的回复

#### 2.2 调度流程

```
用户发送消息 → Orchestrator(Claude Code)分析意图
  → 拆解为子任务1、子任务2、...
  → 并行分派给Agent A、Agent B、...
  → 等待所有结果
  → 聚合输出并在聊天流中汇报
```

#### 2.3 容错能力

- 子Agent执行失败时进行降级处理（如重试、切换Agent、部分结果返回）
- 多Agent产出存在冲突时进行冲突检测和提示

---

### 3. 多Agent接入层（P0）

#### 3.1 架构设计

```
Go后端 ←→ (WebSocket) ←→ 本地守护进程
                              ├── 扫描本地已安装Agent（检测PATH中的CLI命令）
                              ├── 管理Agent生命周期（启动、停止、重启）
                              └── 适配器层（统一Agent输出格式）
```

#### 3.2 Agent发现机制

- 守护进程启动时扫描用户电脑PATH中已知的Agent CLI命令
- 支持的Agent至少包括：Claude Code CLI、Codex CLI / OpenCode
- 发现结果上报给Go后端，在WebUI的Agent列表中展示
- 每个Agent显示为独立的"联系人"，有头像、名称、能力标签

#### 3.3 Agent通信方式

- 所有Agent均为本地安装的CLI工具，不通过远程API调用
- 守护进程通过stdin/stdout与Agent CLI进程通信
- 适配器层负责将不同Agent的输出格式统一为结构化数据

#### 3.4 统一适配器层

每个Agent类型对应一个Adapter，职责：
- 启动和管理CLI进程
- 向CLI传入System Prompt和用户消息
- 解析CLI的stdout输出为结构化产物数据
- 流式转发Agent的中间输出

#### 3.5 自建Agent机制

- 用户在WebUI上创建自定义Agent
- 配置内容：Agent名称、头像、描述、选择底层CLI工具（如Claude Code）、自定义System Prompt
- 本质是对已有CLI工具的个性化包装（自定义角色 + 提示词）
- 创建后出现在Agent列表中，可像其他Agent一样发起对话

---

### 4. 产物预览系统（P1）

#### 4.1 产物数据流

```
Agent CLI输出(stdout) → 适配器解析为结构化产物 → WebSocket推送 → 前端渲染预览卡片
```

#### 4.2 产物类型与渲染

| 产物类型 | 渲染方式 | 说明 |
|----------|----------|------|
| 代码(code) | 语法高亮卡片 | 显示语言标签、文件名、一键复制按钮 |
| 网页(webpage) | iframe预览卡片 | 内嵌预览，点击可展开全屏 |
| 文件(file) | 附件卡片 | 显示文件名、大小，支持下载 |
| Diff(diff) | Diff视图卡片 | 【P2】显示增删改，支持一键应用 |
| 文本(text) | 普通消息气泡 | Agent的普通文字回复 |

#### 4.3 交互能力

- 卡片内联展示在聊天流中
- 点击卡片展开全屏预览
- 代码卡片支持复制代码
- 【P2】支持代码编辑器、Diff应用

---

### 5. 部署发布（P2 — 推迟）

- 聊天中直接发送"部署"指令，Agent返回部署状态卡片
- 一键生成预览URL / 静态站点部署 / 容器化部署 / 源码打包下载

### 6. 多端支持（P2 — 推迟）

| 平台 | 定位 |
|------|------|
| Web端 | 主力端，完整IM体验 + 代码编辑 + 全功能 |
| 桌面端 | 本地文件访问、系统通知、Agent进程管理 |
| 移动端 | 轻量IM体验：查看对话、审批确认、产物预览 |

## 优先级说明

- **P0（必须）**：IM聊天核心体验、单聊/群聊模式、多Agent接入（至少2个）、Orchestrator协调器
- **P1（重要）**：产物预览卡片（结构化渲染）、上下文管理（Pin注入）、多会话并行
- **P2（加分）**：部署发布、Diff视图与版本历史、PPT浏览、多端支持

## 验收标准（Demo演示流程）

Demo需演示以下完整流程：

1. **用户注册登录** — 简单用户名密码，登录后进入IM界面
2. **创建单聊对话** — 选择Agent（如Claude Code），发送任务消息
3. **流式对话体验** — Agent逐字流式回复，消息中展示代码产物卡片（语法高亮）
4. **创建群聊对话** — @多个Agent（如Claude Code + Codex）
5. **Orchestrator调度** — Orchestrator自动拆解任务，并行分派给多个Agent，聚合结果
6. **自建Agent** — 在WebUI上创建自定义Agent（选择CLI工具 + 编写System Prompt）
7. **使用自建Agent** — 用自建Agent发起对话，验证自定义提示词生效

### 可量化的验收检查项

- [ ] 用户可注册、登录，看到自己的对话列表
- [ ] 对话列表支持新建对话、按最近活跃排序
- [ ] 单聊模式下可发送消息并收到Agent流式回复
- [ ] 回复中的代码块以语法高亮卡片形式展示
- [ ] 群聊模式下@多个Agent后，Orchestrator正确分解任务
- [ ] Orchestrator将子任务分派给正确的Agent
- [ ] 至少接入2个Agent平台（如Claude Code + Codex/OpenCode）
- [ ] 本地守护进程能发现已安装的CLI Agent
- [ ] 自建Agent创建后出现在Agent列表中
- [ ] 自建Agent使用自定义System Prompt进行回复
- [ ] Pin消息功能：被Pin的消息内容注入到后续Agent上下文中
- [ ] 多会话并行：可同时开启多个对话窗口
- [ ] 产物卡片可展开全屏预览

## 假设与风险

| 假设/风险 | 说明 | 应对策略 |
|-----------|------|----------|
| Claude Code CLI可用 | Orchestrator依赖本地Claude Code CLI运行 | 守护进程启动时检测，未安装则提示用户 |
| Agent输出格式可解析 | 假设CLI Agent输出Markdown格式，可提取代码块等结构化内容 | 适配器层针对每个Agent做专门的解析逻辑 |
| 本地守护进程稳定性 | 守护进程长时间运行可能出现内存泄漏或断连 | 心跳检测 + 自动重连机制 |
| LLM拆解质量 | Orchestrator的LLM拆解质量直接影响群聊体验 | 设计好Orchestration Prompt，必要时加入示例 |
