# IM 功能增强

> 基于竞品分析（微信/飞书/Slack/Discord）的即时通信必要功能设计。

---

## IM-001 消息编辑

**优先级**: P0  
**描述**: 发送后的消息支持编辑修改，编辑后显示"已编辑"标记。

### 需求

- 文本消息发送后 5 分钟内可编辑（超时不可编辑）
- 编辑后消息尾部显示"已编辑"灰色标记
- 编辑不改变消息在列表中的位置
- 编辑历史不保留（只显示最新版本）
- WS 推送 `message.edited` 事件到对话成员
- 附件消息不可编辑（只能撤回重发）

### 后端

- `PATCH /api/messages/:id` — 编辑消息内容
- 校验：发送者身份、时间窗口、消息类型
- 新增 `edited_at` 字段（nullable timestamp）
- WS 事件：`{ type: "message.edited", data: { message_id, content, edited_at } }`

### 前端

- MessageBubble 长按/右键菜单增加"编辑"选项
- 编辑态：输入框替换消息内容，Enter 确认 / Esc 取消
- 消息气泡底部显示 "已编辑"

## IM-002 消息表情回应（Reaction）

**优先级**: P0  
**描述**: 对消息快速添加 emoji 表情回应，类似 Slack 的 reaction。

### 需求

- 点击消息气泡显示 emoji 快速选择器（常用 6 个 + 更多）
- 同一用户同一消息同一 emoji 只能点一次（toggle）
- 消息底部显示所有 reaction 及数量
- 多人同一 emoji 合并显示头像列表
- WS 实时推送 reaction 变更

### 后端

- `POST /api/messages/:id/reactions` — 添加 reaction `{ emoji: "👍" }`
- `DELETE /api/messages/:id/reactions/:emoji` — 移除 reaction
- 新增 `message_reactions` 表：`id, message_id, user_id, emoji, created_at`
- WS 事件：`{ type: "message.reaction", data: { message_id, emoji, user_id, action: "add"|"remove" } }`

### 前端

- MessageBubble 底部新增 ReactionBar 组件
- 悬停消息显示快速 reaction 工具栏（6 个常用 emoji）
- 点击已有 reaction 切换自己的状态
- 数量 > 0 时显示在消息底部

## IM-003 链接预览卡片

**优先级**: P1  
**描述**: 消息中包含 URL 时自动解析并生成预览卡片。

### 需求

- 检测消息中的 HTTP/HTTPS 链接
- 后端异步抓取链接的 og:title / og:description / og:image
- 生成预览卡片：左侧缩略图 + 右侧标题/描述/域名
- 抓取失败静默降级为普通链接
- 抓取结果缓存（Redis，TTL 24h）

### 后端

- 新增 `link_previews` 表或 Redis 缓存
- 消息发送后异步触发 `fetchLinkPreview(url)`
- 消息 API 返回时附带 `link_preview` 字段

### 前端

- MessageBubble 检测链接后渲染 LinkPreviewCard 组件
- 点击卡片跳转原链接
- 未加载时显示骨架屏

## IM-006 对话免打扰

**优先级**: P1  
**描述**: 单个对话可设置免打扰，不推送通知。

### 需求

- 对话右键菜单"消息免打扰"开关
- 免打扰对话在列表中正常显示但无通知提示
- 免打扰对话不计入未读数
- 设置项持久化到后端

### 后端

- `conversation_members` 新增 `muted` boolean 字段
- `PATCH /api/conversations/:id/mute` — 切换免打扰
- 未读消息计算时排除 muted 对话

### 前端

- ConversationItem 右键菜单增加"免打扰"选项
- 免打扰对话显示静音图标，无未读 badge
- 对话详情面板显示免打扰状态

## IM-007 @提及通知

**优先级**: P0  
**描述**: 在群聊中 @指定成员或 @all，触发高亮显示和定向通知推送。  
**依赖**: MISS-010（@提及功能基础实现）

### 需求

- 输入框 @ 触发成员搜索下拉
- @all 全员提及
- 被 @ 的消息在对话列表显示特殊标记（@ 图标）
- 被 @ 时推送通知（即使免打扰也推送，可选）
- 消息中 @username 高亮显示

### 后端

- 消息解析 @mentions 并存入 `message_mentions` 表
- 通知推送时检查 mentions 表
- `GET /api/conversations/:id/mentions` — 获取未读 @ 消息

## IM-009 群聊禁言控制

**优先级**: P1  
**描述**: 群主/管理员可禁言单个成员或全员禁言。

### 需求

- 成员管理面板：对单个成员"禁言"操作
- 支持禁言时长选择（10 分钟/1 小时/1 天/永久）
- 全员禁言开关（仅管理员可发言）
- 被禁言用户输入框显示"已被禁言"提示

### 后端

- `conversation_members` 新增 `muted_until` nullable timestamp
- `PATCH /api/groups/:id/members/:uid/mute` — 禁言
- `PATCH /api/groups/:id/mute-all` — 全员禁言
- WS chat handler 校验 `muted_until` 拒绝发送
