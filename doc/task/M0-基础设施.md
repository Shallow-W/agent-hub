# M0 项目基础设施

## 目标

初始化三个子项目的工程骨架，建立数据库 schema 和统一 API 规范。

## 子任务

### M0-1 初始化 frontend/

- 用 Vite 创建 React + TypeScript 项目
- 安装核心依赖：react-router, socket.io-client
- 配置 ESLint + Prettier
- 建立 src/ 基础目录结构（components/hooks/services/types/store/utils/styles）

### M0-2 初始化 backend/

- Go module 初始化
- 引入 HTTP 框架（如 Echo 或 Gin）
- 配置管理（config.yaml + 读取逻辑）
- 基础路由注册
- 健康检查接口 `GET /health`

### M0-3 初始化 daemon/

- Go module 初始化
- 创建 main.go 入口
- 建立目录结构（scanner/adapter/process/client）

### M0-4 数据库 Schema

- PostgreSQL 迁移脚本（放 backend/migrations/）
- 核心表：
  - `users`（id, username, password_hash, created_at）
  - `conversations`（id, user_id, type, title, pinned, created_at, updated_at）
  - `conversation_agents`（conversation_id, agent_id）— 群聊关联
  - `messages`（id, conversation_id, role, content, artifacts_json, created_at）
  - `agents`（id, user_id, name, type, cli_tool, system_prompt, avatar, capabilities, created_at）
  - `pinned_messages`（id, conversation_id, message_id, pinned_at）

### M0-5 统一 API 格式

- 成功响应：`{"code": 0, "message": "success", "data": {...}}`
- 错误响应：`{"code": 4xxxx, "message": "具体错误", "data": null}`
- 全局错误处理中间件
- 请求日志中间件

## 验收标准

- [ ] 三个子项目均可独立启动（frontend dev server / backend :8080 / daemon --help）
- [ ] 数据库迁移脚本可执行，表结构符合设计
- [ ] `GET /health` 返回 200
- [ ] API 响应格式统一

## 约束

- frontend 使用 Vite，不用 CRA
- backend 使用 Go 1.22+
- 数据库迁移使用版本化 SQL 文件
