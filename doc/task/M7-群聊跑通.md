# M7 群聊端到端跑通

## 目标

打通群聊完整链路：用户 @多 Agent → Orchestrator 拆解 → 并行调度 → 聚合回显。

## 端到端流程

```
用户输入 "@ClaudeCode @Codex 帮我写一个Todo应用"
  → 前端 WebSocket 发送（conversation.type=group, agent_ids=[...]）
  → Go 后端收到消息，触发 Orchestrator
  → Orchestrator 调用 Claude Code（带 Orchestration Prompt）分析意图
  → 解析出子任务列表：
      子任务1："用React写Todo前端" → Claude Code
      子任务2："用Go写Todo后端API" → Codex
  → 并行分派：通过守护进程同时启动两个 CLI 进程
  → 各 Agent 流式输出 → 后端 WebSocket 推送 → 前端展示
  → 全部完成 → Orchestrator 聚合结果 → 最终汇总消息
```

## 验收标准

- [ ] 用户可在群聊中 @多个 Agent
- [ ] Orchestrator 正确分析意图并拆解任务
- [ ] 子任务被分派给正确的 Agent
- [ ] 多个 Agent 并行执行，结果在聊天流中依次展示
- [ ] 最终有一条汇总消息
- [ ] 某个 Agent 失败时，已完成部分正常展示

## 依赖

- M3-6（@Agent 选择器）
- M4-4 + M4-5（至少两个适配器）
- M5（Orchestrator 全部子任务）
- M6（单聊已跑通，基础链路验证过）
