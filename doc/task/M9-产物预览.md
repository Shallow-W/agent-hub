# M9 产物预览

## 目标

将 Agent 输出中的代码、网页、文件等以结构化卡片形式在聊天流中预览。

## 子任务

### M9-1 适配器输出解析

- 在每个适配器的 StreamChunk 中增加 Artifact 解析
- 从 Agent CLI 的 Markdown 输出中提取：
  - 代码块（```language ... ```）→ `{type: "code", language, filename, content}`
  - URL（http/https 链接）→ `{type: "webpage", url, title}`
  - 文件引用（如 "Created file: xxx"）→ `{type: "file", filename, content}`
- 解析结果附加到 message.artifacts_json 存储

### M9-2 代码卡片组件

- 语法高亮（使用 Prism.js 或 highlight.js）
- 显示：语言标签、文件名（如有）、代码内容
- 操作按钮：复制代码
- 卡片可展开/收起

### M9-3 网页预览卡片

- iframe 内嵌预览（sandbox 安全限制）
- 显示 URL 标题
- 点击展开全屏预览

### M9-4 文件附件卡片

- 显示文件名、文件类型图标
- 下载按钮
- 文件大小显示

## 验收标准

- [x] Agent 返回的代码块自动渲染为语法高亮卡片
- [x] 卡片可展开/收起，代码可一键复制
- [x] 网页链接渲染为 iframe 预览卡片
- [x] 文件附件可下载
- [x] 产物数据持久化，刷新后卡片仍在

## 依赖

- M4-3（适配器接口，在 StreamChunk 中增加 Artifact）
- M6（单聊跑通，有 Agent 回复可解析）

## 优先级

P1 — 单聊群聊跑通后开始

### 实现架构（与原 M9 草案的关键差异）
- **产物存储**：未塞进 `messages.artifacts_json`（该字段已被 agent 元信息占用）。改为**独立 `artifacts` 表**（迁移 023/024/025），关联 `message_id`，含 `root_id`（版本血缘）+ `version`。`artifacts_json` 保持只放 `{agent_id, agent_name, cli_tool}`，零回归。
- **产物解析层**：在**活路径 JS daemon** `src/daemon-npm/bin/agenthub-daemon.js` 的 `parseArtifacts` 做（Go `src/daemon/adapter` 是死代码）。规则：代码块→`code`；`html` 块→`webpage`；`markdown/md/txt/text/json/csv` 块→`document`；散文 URL→`webpage`。
- **跨层契约**：daemon JSON ↔ Go `model.Artifact` json tag ↔ 前端 TS `Artifact`，字段 `type/language/filename/title/url/content/root_id/version` 五层对齐。

### 产物类型与卡片
| 类型 | 触发 | 卡片/预览 |
|------|------|-----------|
| code | 代码块 | 内联高亮+复制+展开；全屏 CodeMirror 编辑器 |
| webpage | URL / html 块 | sandbox iframe 缩略 + 全屏 Preview |
| document | markdown/json/csv 块 | 文档卡 + 全屏渲染（markdown 复选框可交互打勾）|
| file | — | 文件卡 + 下载 |
| PPT[P2] | 用户上传 .pptx 附件 | pptx-preview 浏览器内全屏渲染（真实比例、懒加载）|

### P2 能力
- **代码编辑器**：CodeMirror 6 全屏「查看/编辑」+ 复制 + 下载（懒加载）。
- **版本历史**：独立 artifacts 表 `root_id` 血缘 + `version` 递增；全屏版本下拉切换 / 查看；旧版「回滚到此版本」（生成新最新版，保留全部历史）。后端 `GET/POST /api/artifacts/:rootId/versions`（成员鉴权）。
- **Diff 视图**：`@codemirror/merge` 双栏只读对比任意两版本，改动行高亮、完整展开（懒加载）。
- **对话式局部修改（旗舰）**：专用入口方案——全屏代码里选中片段（CodeMirror `drawSelection` 持久蓝色高亮 + 选完才捕获 + 「全选」按钮 + 选区预览）→ 输入指令 → `POST /api/artifacts/:rootId/ai-edit` 派一次性 daemon 任务（复用 orchestrator dispatch+waitDaemonTask）→ 提取修改后代码 → 存为新版本（不污染聊天流）。

### 关键 bug 修复记录
1. **root_id 跨层贯通**：① `ListByMessageIDs` 外层子查询致 pgx uuid/timestamptz 扫描丢空（改直查 `DISTINCT ON` + Go 排序）；② WS 广播缺 root_id（`CreateArtifacts` 回填内存对象）。
2. **前端渲染副作用**：MessageBubble 内容→root_id 匹配用渲染期 mutation 的 `consumed` Set，在 React StrictMode 双调用下失效（改纯函数 `buildContentRootMap` Map 查表）。
3. **上传/拖拽**：upload.go 白名单加 pptx/office/text（扩展名 gate + zip/octet-stream 兜底 + 修 charset mime）；ChatWindow window 级拖拽兜底 + 整窗 drop 区。
4. **PPT 预览**：从真实 `previewer.pptx` 读宽高比，4:3/16:9 都不裁切、居中、大背景。
5. **选区体验**：实时捕获改为松手/抬键后捕获（+debounce）。
