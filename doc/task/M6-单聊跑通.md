# M6 单聊端到端跑通

## 目标

打通单聊完整链路：用户发消息 → 后端 → 守护进程 → Agent CLI → 流式回显。

## 端到端流程

```
用户输入 → 前端 WebSocket 发送
  → Go 后端收到消息，创建 message 记录
  → 后端将任务通过 WebSocket 转发给守护进程
  → 守护进程通过适配器启动 Agent CLI（如 claude）
  → CLI 流式输出 → 适配器解析 → 守护进程转发 → 后端 → WebSocket → 前端
  → 前端逐字渲染
  → CLI 完成 → 适配器返回最终结果（含结构化产物）→ 存储到 message.artifacts_json
```

## 验收标准

- [ ] 用户可选择一个 Agent（如 Claude Code）开始单聊
- [ ] 发送消息后，Agent 逐字流式回复
- [ ] 回复中的代码块以语法高亮形式展示
- [ ] 消息持久化，刷新页面后历史消息仍在
- [ ] 连续多轮对话，上下文正确传递给 Agent

## 依赖

- M3（聊天 UI + 消息收发）
- M4（Agent 接入 + 适配器）

## 注意事项

- 这是第一个全栈端到端跑通点，可能暴露各层集成问题
- 优先保证链路通，再优化体验
