# 优化聊天前端 UI

## 背景

当前 Web UI 已具备 AgentHub 的基础聊天功能，但视觉密度、IM 桌面端布局、顶部工具栏和消息流呈现与目标参考图存在差距。需要在不改动后端接口的前提下，优化前端布局和样式，使界面更接近参考图的轻量 IM 协作工作台。

## 目标

- 左侧导航更接近参考图：窄图标栏优先，品牌与状态信息弱化，整体更轻。
- 会话列表提升桌面 IM 质感：更紧凑的标题栏、搜索框、选中态和列表项。
- 聊天区更接近参考图：白色顶部栏、浅灰网格背景、消息内容居中约束、输入区固定底部。
- 消息气泡更适合多 Agent 协作：用户消息轻蓝色，Agent 消息白底卡片化，长内容可读性更好。
- 保持现有功能：对话、好友、群聊、搜索、附件、回复、输入和 WebSocket 状态不回退。

## 非目标

- 不新增后端字段或接口。
- 不重构状态管理。
- 不实现任务看板或产物预览的新业务能力。

## 验收标准

- `npm run build` 通过。
- 主聊天页在桌面宽度下接近参考图的信息结构和视觉节奏。
- 前端代码继续使用 CSS Modules，避免新增 `any` 和组件内 API 直连。
