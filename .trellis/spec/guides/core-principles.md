# Core Principles

> AI 行为准则——每次任务前必须遵守的 4 条核心规则。

---

## 规则 1：先想后写
杜绝隐蔽假设。明确声明你的前提假设，阐明方案权衡。有疑问先确认，切忌盲目猜测。若存在更简单的实现路径，应果断提出异议。

## 规则 2：简单至上
仅以最少代码解决问题。不添加"以防万一"的冗余功能，不为仅用一次的代码强行设计抽象层。若资深工程师会认为其过度复杂，则立即简化。

## 规则 3：外科手术式修改
仅改动绝对必要的部分。切勿"顺手优化"相邻代码、注释或格式排版。未出问题的代码绝不重构。严格贴合项目现有代码风格。

## 规则 4：目标驱动
明确定义成功标准（验收条件）。持续迭代直至验证通过。不要规定具体操作步骤，只需清晰描述"成功的最终形态"，交由模型自主探索与迭代。
